## Scaling Principles

1. **Core API is stateless** вҖ” all state in PostgreSQL + Redis. Any instance can handle any request.
2. **WebSocket sessions** вҖ” sticky sessions via Traefik `cookie` LB or shared Redis pub/sub for broadcasts.
3. **Keeper** вҖ” one per Core API instance (sidecar) or shared pool behind internal LB.
4. **Ollama** вҖ” replica set with round-robin or GPU-aware scheduling.
5. **Skills** вҖ” independently scalable microservices; Core API routes via service discovery.
